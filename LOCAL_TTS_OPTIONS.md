# 本地情感 TTS 方案对比

考虑到你的核心需求：
1. **情感表现力** (Support emotions)
2. **本地部署** (Local)
3. **低配置要求** (Low Resource)
4. **低延迟** (Low Latency)

以下是三个最佳候选方案的深度对比：

| 特性 | **GPT-SoVITS** (推荐) | **VITS (Coqui/MoeGoe)** | **ChatTTS** |
| :--- | :--- | :--- | :--- |
| **情感表现力** | ⭐⭐⭐⭐⭐ (极强)<br>可模仿任何语气，通过参考音频控制 | ⭐⭐⭐ (中等)<br>基于预设模型，情感较为固定 | ⭐⭐⭐⭐ (强)<br>非常自然，但情感控制不如 Sovits 精确 |
| **配置要求** | 🟡 **中等**<br>显存: 4GB+ (推荐 6GB)<br>内存: 8GB+ | 🟢 **极低**<br>显存: 2GB+ (甚至 CPU 可行)<br>内存: 4GB+ | 🔴 **高**<br>显存: 4GB+ (推荐 8GB)<br>内存: 16GB+ |
| **延迟** | 🟡 **中等**<br>首字延迟: ~0.5-1秒 (优化后) | 🟢 **极低**<br>首字延迟: <0.2秒 (即时) | 🔴 **高**<br>生成较慢，适合非实时 |
| **声音克隆** | ✅ **支持** (5秒音频即可) | ❌ 需重新训练模型 | ❌ 不支持快速克隆 |
| **适用场景** | **情感对话、角色扮演** | 快速响应、低配设备 | 长文本朗读、有声书 |

---

## 详细分析

### 1. GPT-SoVITS (综合最佳) 🏆
目前最流行的开源方案。它引入了"参考音频"机制，你只需要提供一段 5-10 秒带有特定情感的音频（如"生气的Hiyori"），它就能用同样的语气说出任何话。

*   **为什么适合你**：显存要求适中（你的配置能跑 Whisper large-v3，肯定能跑这个），情感表现力是目前天花板级别。
*   **缺点**：相比 Edge TTS，会有约 0.5-0.8 秒的计算延迟（可以通过流式输出优化）。

### 2. VITS (速度之王) 🚀
轻量级方案，速度极快，甚至可以在 CPU 上跑。

*   **为什么适合你**：如果你极致追求"不卡顿"和"低延迟"。
*   **缺点**：情感通常是训练在模型里的（如"开心版模型"、"生气版模型"），不能像 GPT-SoVITS 那样灵活地通过音频控制情感。声音听起来可能比 Sovits 稍显机械。

### 3. ChatTTS (自然度高) 🗣️
专门针对"对话自然度"优化，会自动加笑声、停顿、口语词。

*   **为什么不推荐**：它更侧重于"像真人说话"（甚至有点啰嗦），而不是"精确的情感控制"。而且它对配置要求较高，且目前推理速度较慢。

---

## 💡 我的建议

**方案 A：追求极致情感 (推荐)**
👉 选择 **GPT-SoVITS**
*   理由：既然已经因为 Edge TTS "没感情" 而放弃，那就应该选情感最强的。你的电脑能跑 Whisper，跑这个没问题。
*   工作流：准备几个文件夹（`happy/`, `sad/`, `angry/`），里面放几句对应情感的音频，TTS 自动调用。

**方案 B：追求极致速度**
👉 选择 **VITS**
*   理由：如果你的电脑显卡比较老（如 GTX 1060 以下），或者你无法忍受 0.5 秒的延迟。

**方案 C：混合架构**
*   **日常回复**：用 VITS (秒回)
*   **情感回复**：用 GPT-SoVITS (带感情)
